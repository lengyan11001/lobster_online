# 模板定制技能：服务器模板列表与调用参数

本文说明 online 客户端里“模板定制”能力如何从服务器读取模板列表，以及其他 online 功能复用模板定制时应该怎么传参数。

## 1. 调用关系

模板配置以服务器为准，本地不写死模板列表。

推荐调用链：

```text
其他 online 功能 / OpenClaw
  -> invoke_capability(capability_id="cutcli.template.customize", payload={...})
  -> 本机 MCP
  -> 本机后端 /api/cutcli/local/capability
  -> 需要模板配置时转发到服务器 /api/cutcli/templates
  -> 本机按服务器模板执行：取素材 -> 提取音频 -> 服务器 STT -> 本机 ffmpeg 或 CutCLI 云渲染
```

页面直接调用链：

```text
模板定制页面
  -> GET /api/cutcli/local/templates
  -> 本机后端转发 GET {AUTH_SERVER_BASE}/api/cutcli/templates
```

## 2. 鉴权约定

本机接口和服务器接口都需要当前登录用户的 Bearer Token。

前端调用本机接口时使用现有登录态即可：

```http
Authorization: Bearer <user_jwt>
X-Installation-Id: <installation_id 可选>
```

本机后端会把 `Authorization` 和 `X-Installation-Id` 转发给服务器模板接口。业务方不要单独保存服务器 token。

## 3. 获取服务器模板列表

### 3.1 页面/本机后端调用

```http
GET /api/cutcli/local/templates
Authorization: Bearer <user_jwt>
```

本机后端实际会请求：

```http
GET {AUTH_SERVER_BASE}/api/cutcli/templates
Authorization: Bearer <user_jwt>
```

### 3.2 能力调用方式

其他功能推荐通过能力调用查模板：

```json
{
  "capability_id": "cutcli.template.customize",
  "payload": {
    "action": "templates"
  }
}
```

返回体就是服务器模板列表。

## 4. 模板列表返回字段

返回示例：

```json
{
  "ok": true,
  "templates": [
    {
      "id": "auto_caption_neon_focus_v1",
      "kind": "auto_caption",
      "name": "霓虹聚焦字幕",
      "description": "模板说明",
      "aspect_ratio": "source",
      "tags": ["口播", "字幕"],
      "quality_label": "样式标签",
      "input_modes": ["upload", "asset_id", "video_url"],
      "preserve_source_video": true,
      "preview_url": "/client/client-code/cutcli_templates/auto_caption_neon_focus_v1.mp4",
      "sample_video_url": "/client/client-code/cutcli_templates/auto_caption_neon_focus_v1.mp4",
      "render_path": "/api/cutcli/templates/auto_caption_neon_focus_v1/render",
      "render_modes": ["ffmpeg", "cutcli_cloud"],
      "overlay_fields": [
        {
          "key": "title",
          "label": "标题",
          "max_length": 24
        }
      ],
      "caption_style": {},
      "generation_strategy": {
        "executor": "online",
        "stt": {
          "provider": "sutui",
          "model": "volcengine/speech-to-text/bigmodel-v2",
          "server_endpoint": "/api/cutcli/stt/transcribe",
          "input": "audio_url"
        },
        "cloud_render_endpoint": "/api/cutcli/cloud/render-draft",
        "render_modes": ["ffmpeg", "cutcli_cloud"],
        "overlay_fields": []
      }
    }
  ]
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `id` | 模板 ID，提交生成时传 `template_id`。 |
| `name` | 展示名称。 |
| `description` | 展示说明。 |
| `aspect_ratio` | 模板比例。`source` 表示保留原视频比例。 |
| `input_modes` | 支持的输入方式，一般包括 `upload`、`asset_id`、`video_url`。 |
| `preview_url` / `sample_video_url` | 样片地址，前端预览用。相对路径时按服务器域名拼接。 |
| `render_modes` | 支持的渲染方式。`ffmpeg` 是本机渲染，`cutcli_cloud` 是云渲染。 |
| `overlay_fields` | 该模板允许业务传入的叠加文案字段。 |
| `caption_style` | 字幕样式。业务方只展示或透传，不要本地改写。 |
| `generation_strategy` | 执行策略。业务方只读，不要在客户端写死。 |

## 5. 提交模板定制任务

其他 online 功能推荐调用能力，不直接拼页面接口。

```json
{
  "capability_id": "cutcli.template.customize",
  "payload": {
    "action": "start",
    "template_id": "auto_caption_neon_focus_v1",
    "render_mode": "ffmpeg",
    "asset_id": "776f7ae646b5",
    "overlay_texts": {
      "title": "门店爆款活动",
      "subtitle": "扫码了解福利"
    },
    "position_overrides": {
      "caption": {
        "x": 0,
        "y": -0.72
      },
      "overlay": {
        "title": {
          "x_ratio": 0.5,
          "y_ratio": 0.18
        }
      }
    },
    "external_task_id": "caller_job_001"
  }
}
```

### 5.1 `payload` 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `action` | 是 | `start` 提交任务；`poll` 查任务；`list` 查历史；`templates` 查模板。 |
| `template_id` | `start` 必填 | 模板 ID，来自服务器模板列表。 |
| `render_mode` | 否 | `ffmpeg` 或 `cutcli_cloud`，默认 `ffmpeg`。必须是模板 `render_modes` 支持的值。 |
| `asset_id` | 二选一 | 素材库视频 ID。 |
| `video_url` | 二选一 | 公网可访问视频 URL。 |
| `overlay_texts` | 否 | 模板叠加文案，字段以模板 `overlay_fields` 为准。 |
| `position_overrides` | 否 | 字幕或叠加文案位置微调。 |
| `external_task_id` | 否 | 调用方自己的任务 ID，用于关联。 |

`asset_id` 和 `video_url` 二选一。优先用 `asset_id`，这样生成结果和素材库记录能关联到当前用户。

### 5.2 `overlay_texts`

只传服务器模板允许的字段。常见字段：

```json
{
  "top_text": "顶部主标题",
  "title": "标题",
  "subtitle": "副标题",
  "badge": "角标",
  "headline": "兼容旧字段：主标题",
  "subheadline": "兼容旧字段：副标题"
}
```

本机后端会按模板 `overlay_fields.max_length` 截断，未允许的字段会被忽略。

### 5.3 `position_overrides`

结构示例：

```json
{
  "caption": {
    "x": 0,
    "y": -0.72
  },
  "overlay": {
    "title": {
      "x_ratio": 0.5,
      "y_ratio": 0.18
    },
    "subtitle": {
      "x_ratio": 0.5,
      "y_ratio": 0.28
    },
    "badge": {
      "x_ratio": 0.5,
      "y_ratio": 0.62
    }
  }
}
```

取值范围：

| 字段 | 范围 | 说明 |
| --- | --- | --- |
| `caption.x` / `caption.y` | `-0.95` 到 `0.95` | 字幕归一化坐标。`x=0` 居中；`y` 越小越靠下，越大越靠上。 |
| `overlay.*.x_ratio` / `overlay.*.y_ratio` | `0.03` 到 `0.97` | 画面比例坐标。左上为接近 `0`，右下为接近 `1`。 |

## 6. 查询任务

提交成功示例：

```json
{
  "capability_id": "cutcli.template.customize",
  "result": {
    "ok": true,
    "job_id": "20260613010101_abcdef12",
    "status": "queued",
    "stage": "queued",
    "template_id": "auto_caption_neon_focus_v1",
    "poll_path": "/api/cutcli/local/templates/jobs/20260613010101_abcdef12"
  }
}
```

轮询：

```json
{
  "capability_id": "cutcli.template.customize",
  "payload": {
    "action": "poll",
    "job_id": "20260613010101_abcdef12"
  }
}
```

也可以直接调本机接口：

```http
GET /api/cutcli/local/templates/jobs/20260613010101_abcdef12
Authorization: Bearer <user_jwt>
```

完成示例：

```json
{
  "ok": true,
  "job_id": "20260613010101_abcdef12",
  "status": "completed",
  "stage": "completed",
  "preview_asset_id": "abc123def456",
  "final_asset_id": "abc123def456",
  "preview_url": "/api/assets/file/abc123def456?token=...",
  "open_url": "/api/assets/file/abc123def456?token=...",
  "caption_count": 18,
  "render_strategy": "local_ffmpeg",
  "error": ""
}
```

失败示例：

```json
{
  "ok": true,
  "job_id": "20260613010101_abcdef12",
  "status": "failed",
  "stage": "failed",
  "error_code": "cutcli_local_failed",
  "failed_stage": "render",
  "error": "ffmpeg stderr tail..."
}
```

`failed_stage` 常见值：

| 阶段 | 说明 |
| --- | --- |
| `resolve_source` | 解析素材或视频 URL。 |
| `extract_audio` | 本机 ffmpeg 提取音频。 |
| `stt` | 调服务器 STT。 |
| `render` | 本机 ffmpeg 或 CutCLI 云渲染。 |

## 7. 查询历史

能力调用：

```json
{
  "capability_id": "cutcli.template.customize",
  "payload": {
    "action": "list",
    "limit": 20
  }
}
```

本机接口：

```http
GET /api/cutcli/local/templates/jobs?limit=20
Authorization: Bearer <user_jwt>
```

## 8. 页面表单接口

模板定制页面仍可使用原表单接口：

```http
POST /api/cutcli/local/templates/{template_id}/render
Authorization: Bearer <user_jwt>
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `file` | 三选一 | 上传视频文件。 |
| `asset_id` | 三选一 | 素材库视频 ID。 |
| `video_url` | 三选一 | 公网视频 URL。 |
| `render_mode` | 否 | `ffmpeg` 或 `cutcli_cloud`。 |
| `position_overrides` | 否 | JSON 字符串。 |
| `top_text` / `title` / `subtitle` / `headline` / `subheadline` / `badge` | 否 | 叠加文案字段。 |

示例：

```bash
curl -X POST "http://127.0.0.1:8000/api/cutcli/local/templates/auto_caption_neon_focus_v1/render" \
  -H "Authorization: Bearer $TOKEN" \
  -F "asset_id=776f7ae646b5" \
  -F "render_mode=ffmpeg" \
  -F "title=门店爆款活动" \
  -F "subtitle=扫码了解福利" \
  -F 'position_overrides={"caption":{"x":0,"y":-0.72}}'
```

## 9. 接入规则

- 模板列表、模板字段、样片地址都以服务器 `/api/cutcli/templates` 返回为准。
- 其他功能不要本地写死模板配置，只保存用户选择的 `template_id` 和业务输入。
- 提交任务时不需要传 `caption_style` 或 `generation_strategy`，本机会按 `template_id` 重新向服务器取最新配置。
- 如果服务器模板字段变更，客户端下一次拉模板即可生效。
- 生成结果会入本机素材库，返回 `preview_asset_id` / `final_asset_id`。
- 业务方展示视频优先使用 `open_url`，其次使用 `preview_url`。
