# 生产环境：完整依赖包（干净 zip）

目标：产出 **仅含运行所需** 的 Windows 离线安装包，**不包含** 开发文档、历史压缩包、本机密钥与制包机私货。

---

## 三种交付包对照（完整依赖 / 无大依赖安装 / 代码更新）

| 类型 | 用途 | 常用脚本 | 用户侧安装 / 更新 |
|------|------|----------|-------------------|
| **① 完整依赖包** | 新装、离线或尽量少联网；自带/捆绑 Python、wheels、Node、浏览器等大件 | `scripts/build_result_package.sh` 一键链；或依赖已齐时 `SKIP_ENSURE_FULL_PACK_DEPS=1 bash pack_full_project.sh`；Windows 无 bash 可用 `scripts/pack_full_project_zip.py` | 解压后运行 **`install.bat`** |
| **② 无大依赖安装包（Slim）** | 包体小，**不含** `python/`、`nodejs/`、`deps/` 等；**首次安装需联网**拉运行时与 pip/npm/Playwright 等 | **`python scripts/pack_slim_zip.py`**（产物在 `lobster_online` **上一级**目录：`lobster_online_slim_<品牌>_<时间戳>.zip`） | 解压后运行 **`install_slim.bat`**，说明见包内 **`README_SLIM.txt`**（原「使用说明-精简包」）；一键安装并启动用 **`quick_start.bat`** |
| **②′ 纯代码包（更小）** | 仅源码与清单，**不含** `deps/wheels`、不含 `ffmpeg.exe`；适合已有 Python 或配合 Slim 思路 | **`bash pack_code_only.sh`**（需 bash + zip）；Windows 未装 zip 时与 sh 同清单：**`python scripts/pack_code_only_zip.py`**（产物仍在 `lobster_online/` 根目录 `lobster_online_code_only_*.zip`） | 目标机自行 **`pip install -r requirements.txt`** 或配合已有环境 |
| **③ 代码更新包** | 已部署机器**只覆盖代码**，不动 `python/`、`deps/`、`browser_chromium/`、`nodejs` 可执行文件等 | **`bash pack_code.sh`**（产物在 `lobster_online/` 下 `lobster_online_code_*.zip`；公网勿带 `pack_bundle.env` 时用 `SKIP_PACK_BUNDLE_ENV=1`）；OTA/热更新与 `check_client_code_update.py` 路径一致时用 **`python scripts/pack_client_code_ota.py`**（默认 zip 在 **`lobster_online` 上一级**，与 Slim 包同级；可选 `--out`） | 覆盖解压或按 OTA manifest；**不替代**首次完整/Slim 安装 |

**记忆口诀**：完整 → `install.bat`；无大依赖（Slim）→ `install_slim.bat`；更新 → `pack_code` / `pack_client_code_ota`，不替换运行时。

---

## 一键流程（推荐）

在 **`lobster_online/`** 目录执行：

```bash
bash scripts/build_result_package.sh
```

顺序为：

1. `report_pack_gaps.py`（只读，看缺口）
2. `build_package.sh`（嵌入式 Python、Node、get-pip、wheels、OpenClaw 等，需联网）
3. 下载 `deps/vc_redist.x64.exe`（若尚无）
4. `ensure_full_pack_deps.sh`（补齐 `deps/wheels` 与离线校验）
5. `browser_chromium`（非 Windows 且目录空时会报错退出，需先在 Windows 生成再拷回）
6. `report_pack_gaps.py`（复检，**不齐则中止、不生成 zip**）
7. `pack_full_project.sh`（**仅 zip**，不改 `install.bat` / `start.bat` / `run_*.bat`）

产物：上一级目录 **`lobster_online_完整项目包_*.zip`**。

---

## 依赖已齐、只打 zip

```bash
cd lobster_online
SKIP_ENSURE_FULL_PACK_DEPS=1 bash pack_full_project.sh
```

或旧版带「使用说明.txt」的完整包（体积与排除项略不同）：

```bash
SKIP_ENSURE_FULL_PACK_DEPS=1 bash pack_full.sh
```

---

## 只拉依赖、不打包

```bash
bash scripts/sync_deps_for_pack.sh
```

## 上线包默认不包含的内容（由 `pack_full_project.sh` 排除）

| 类别 | 说明 |
|------|------|
| 密钥与本地配置 | 根目录 **`.env`**、**`pack_bundle.env`**（制包/开发用；勿把个人测试 Key 打进公网包） |
| 文档 | **`docs/`** 下全部、`README.md` 等根目录说明性 **`.md`**（运行时不需要） |
| 产物与缓存 | `*.db`、`*.pyc`、`__pycache__`、`**/*.zip`（根目录）、`installed_packages.json`、`mcp_registry_cache.json`、`logs/` |
| 用户数据目录 | `openclaw/workspace/`、`browser_data/`、`assets/`（运行时生成） |
| 制包脚本 | `pack_*.sh`、`build_package.sh`、`scripts/ensure_*pack*.sh`、`scripts/build_result_package.sh` 等 |

用户首次安装：**`install.bat`** 在缺少 `.env` 时会从 **`.env.example`** 复制为 `.env**；请在目标环境自行填写 **`SECRET_KEY`**、速推/模型等密钥，**不要**把个人测试用 `.env` 直接打进 zip。

## 与「代码增量包」的区别

| 脚本 | 用途 |
|------|------|
| `pack_full_project.sh` | 完整依赖 + 运行环境，适合新装用户 |
| `pack_code.sh` | 仅代码 + `deps/wheels`，不含 `python/`、`nodejs/` 大目录；默认会把 **`pack_bundle.env` 打成包内 `.env`**（制包机自用） |

公网交付、**不要带制包机里的 `pack_bundle.env`** 时：

```bash
SKIP_PACK_BUNDLE_ENV=1 bash pack_code.sh
```

更细的 install 行为见 **`docs/离线安装依赖清单.md`**（路径在源码仓；打进 `pack_full_project` 的 zip **不含** `docs/`）。
