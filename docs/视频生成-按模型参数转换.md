# 图片/视频生成：按模型传不同参数，谁来做

## 结论

- **不同模型 API 参数确实不同**（如图片 flux 用 `image_urls` 数组、jimeng 用 `image_url`；视频 Seedance 用 `ratio`/`filePaths`/`functionMode`，wan 用 `duration` 字符串等）。
- **由我们（lobster_online MCP）做这层转换**：在调用上游前，根据 `payload.model` 把统一 payload 转成该模型对应的参数格式，并**保证用户输入的 prompt 原样传入**，不改写、不臆造。

## 谁来做

| 方案 | 说明 |
|------|------|
| **我们（MCP）做** | 在 `mcp/http_server.py` 的 `invoke_capability` 里，对 `image.generate` 调用 `_normalize_image_generate_payload(payload)`，对 `video.generate` 调用 `_normalize_video_generate_payload(payload)`，再调上游。 |

## 代码位置

- **图片**：`_normalize_image_generate_payload(payload)`，当 `capability_id == "image.generate"` 时执行。
- **视频**：`_normalize_video_generate_payload(payload)`，当 `capability_id == "video.generate"` 时执行。

## 已支持图片模型（参数映射 + prompt 透传）

| 模型 | 映射说明 |
|------|----------|
| jimeng-4.0 / jimeng-4.5 | prompt, image_url（可选）, n |
| fal-ai/flux-2/flash | prompt, image_urls（由 image_url 转数组）, image_size, num_images |
| fal-ai/bytedance/seedream/* | prompt, image_size, num_images |

## 已支持视频模型（参数映射 + prompt 透传）

### 统一 UI 字段与上游枚举（避免 422）

速推/新版 UI 常见字段：

- **`aspect_ratio: "auto"`**：MCP 会规范为 **`16:9`**（上游只接受 `21:9`、`16:9`、`4:3`、`1:1`、`3:4`、`9:16` 等枚举，**不接受** `auto`，否则会 **HTTP 422**）。
- **`duration: "6s"`** 或 **`6`**：会解析为秒数整数，再按各模型要求输出（如 Seedance 顶层 `duration` 字符串秒数、Veo 的 `4s`/`6s`/`8s`）。**禁止**把未解析的 `"6s"` 直接传给只接受 `int(...)` 的旧分支，否则本地即抛错或上游 422。

### 模型名称映射（展示名 → 标准 ID）

系统会自动将常见的模型展示名（如 "Sora 2 Pub"）映射为标准模型 ID（如 `fal-ai/sora-2/image-to-video`），确保 LLM 即使使用展示名也能正确调用模型。

**支持的映射规则**：
- **Sora 2 系列**：
  - "Sora 2 Pub" / "sora2pub" → `fal-ai/sora-2/image-to-video`（有图片）或 `fal-ai/sora-2/text-to-video`（无图片）
  - "Sora 2 VIP" / "sora2vip" → `fal-ai/sora-2/vip/image-to-video` 或 `fal-ai/sora-2/vip/text-to-video`
  - "Sora 2 Pro" / "sora2pro" → `fal-ai/sora-2/pro/image-to-video` 或 `fal-ai/sora-2/pro/text-to-video`
- **Seedance 系列**：
  - "Seedance 2" / "Seedance 2.0" → `st-ai/super-seed2`
  - "Seedance 1.5" → `fal-ai/bytedance/seedance/v1.5/pro/image-to-video` 或 `text-to-video`
  - "Seedance 1" / "Seedance v1" → `fal-ai/bytedance/seedance/v1/pro/image-to-video` 或 `text-to-video`
- **Kling 系列**：自动映射到 `fal-ai/kling-video/*` 系列
- **Wan 系列**：自动映射到 `wan/v2.6/image-to-video` 或 `wan/v2.6/text-to-video`
- **Veo 系列**：自动映射到 `fal-ai/veo3.1/image-to-video` 或 `fal-ai/veo3.1`（文生）；**勿默认走** `google/veo-3.1/*`（速推侧常需等候名单）
- **Grok 系列**：自动映射到 `xai/grok-imagine-video/image-to-video` 或 `xai/grok-imagine-video/text-to-video`

**注意**：如果传入的已经是标准模型 ID（如 `fal-ai/sora-2/image-to-video`），系统会直接透传，不会再次映射。

### 通用处理模型（自动适配参数）

以下模型系列使用通用处理逻辑，**自动适配**图生视频（i2v）和文生视频（t2v）参数：

| 模型系列 | 说明 |
|------|----------|
| **Sora 2 系列** | sora-2/pub, sora-2/vip, sora-2/pro, fal-ai/sora-2/* |
| **Kling 系列** | kling-video, kling-o3, kling-v3, fal-ai/kling-video/* |
| **Veo 3.1 系列** | veo-3.1, veo-3.1-fast, fal-ai/veo3.1/*（口语「Veo」默认此路径）；显式 `google/veo-3.1/*` 仍透传 | duration 必须是字符串格式：'4s', '6s' 或 '8s'（默认 '6s'） |
| **Grok Imagine Video** | grok-imagine-video, xai/grok-imagine-video/* |
| **即梦系列** | jimeng, 即梦-3.5-pro, 即梦/* |
| **其他所有模型** | 自动适配：i2v 传 image_url，t2v 传 aspect_ratio，保留所有用户参数 |

### 通用处理规则

对于所有模型（包括未列出的），系统会：

1. **图生视频（i2v）**：自动传递 `image_url`（从 `filePaths`/`media_files`/`image_url` 中提取）
2. **文生视频（t2v）**：自动传递 `aspect_ratio`（默认 16:9）
3. **保留所有参数**：用户传入的 `duration`、`resolution`、`audio`、`seed`、`negative_prompt` 等参数会原样传递
4. **Prompt 透传**：用户输入的 prompt 原样传递，不做改写
5. **Options 对象处理**：
   - **Seedance v1/v1.5 系列**：自动将 `resolution`、`generate_audio`、`camera_fixed`、`seed`、`end_image_url`、`reference_image_urls` 等参数放入 `options` 对象
   - **其他模型**：参数在顶层，但用户可直接传入 `options` 对象，系统会保留

**因此，用户可以直接指定任意速推支持的模型名称，系统会自动适配参数格式。**

### 重要：Seedance 系列的 options 对象

**Seedance v1.5 和 v1 系列**（如 `fal-ai/bytedance/seedance/v1.5/pro/text-to-video`）使用特殊的参数结构：

- **顶层参数**：`prompt`、`duration`（字符串）、`image_url`（i2v 时）、`aspect_ratio`
- **`options` 对象**：包含 `resolution`、`generate_audio`、`camera_fixed`、`seed`、`end_image_url`、`reference_image_urls`、`enable_safety_checker` 等

**示例**：
```json
{
  "model": "fal-ai/bytedance/seedance/v1.5/pro/image-to-video",
  "prompt": "A man is crying",
  "image_url": "https://...",
  "duration": "5",
  "options": {
    "resolution": "720p",
    "generate_audio": true,
    "camera_fixed": false
  }
}
```

系统会自动将用户传入的 `resolution`、`generate_audio` 等参数放入 `options` 对象，无需手动构造。

## 对话侧 prompt 规则（chat 系统提示）

- **图片与视频统一**：默认用用户原话作 prompt（先去掉「发布到某账号」等指令性表述），不要改写、不要根据图片臆造内容；**仅当用户明确说「让ai写词」「你写词」「帮我写提示词」时**，才由 AI 撰写 prompt。

## 扩展新模型

在 `_normalize_image_generate_payload` 或 `_normalize_video_generate_payload` 里按 `payload.get("model")` 增加分支，构造该模型 API 所需的参数字段即可（可参考 xskill 对应模型的 llms.txt）。
