# 上传流程日志追踪说明

本文档说明从用户点击上传到服务器处理完成的完整流程，每一步都有详细的日志标记。

## 流程概览

### 客户端上传阶段

#### 步骤1：客户端接收上传请求
- **位置**: `lobster_online/backend/app/api/assets.py` - `upload_asset()`
- **日志标记**: `[上传流程-步骤1]`
- **内容**: 记录文件名、用户ID、文件大小、媒体类型

#### 步骤2：尝试TOS上传
- **位置**: `lobster_online/backend/app/api/assets.py` - `_upload_to_tos()`
- **日志标记**: `[TOS-步骤2.1]` 到 `[TOS-步骤2.5]`
- **子步骤**:
  - 2.1: 检查TOS配置
  - 2.2: 验证配置完整性
  - 2.3: 创建TOS客户端
  - 2.4: 执行上传
  - 2.5: 返回结果

#### 步骤3：如果TOS失败，上传到服务器临时文件接口
- **位置**: `lobster_online/backend/app/api/assets.py` - `upload_asset()`
- **日志标记**: `[上传流程-步骤3]`
- **子步骤**:
  - 3.1: 获取服务器地址
  - 3.2: 获取认证token
  - 3.3: 发送HTTP请求
  - 3.4: 处理响应

#### 服务器端接收临时文件（步骤3.1-3.5）
- **位置**: `lobster-server/backend/app/api/assets.py` - `upload_temp_file()`
- **日志标记**: `[服务器端-步骤3.1]` 到 `[服务器端-步骤3.5]`

#### 服务器直连上传 `/api/assets/upload`（非 lobster_online 客户端）
- **位置**: `lobster-server/backend/app/api/assets.py` - `upload_asset()`
- **规则**: **仅 TOS 成功才入库**；无 TOS 公网 URL 时打 **`[上传流程-失败]`**，删本地临时文件并 **HTTP 503**（与 online 客户端「双失败则报错」一致）。
- **`get_asset_public_url`**: 必须使用 **`Asset.asset_id`** 查询（不得误用整数主键 `Asset.id`）；无公网 `source_url` 时返回 **None**，**不再**回退 `build_asset_file_url`。

#### 服务器对话 `POST /chat`（兼容路径）
- **位置**: `lobster-server/backend/app/api/chat.py`
- **规则**: 与 `lobster_online` 一致 — `[使用素材-步骤B.*]`、`[使用素材-失败]`；**禁止** `/api/assets/file/`；**补传 `attachment_urls`** 给 `_chat_openai` / `_chat_anthropic`（此前同步 `/chat` 未传导致图生视频注入为空）。
- **子步骤**:
  - 3.1: 接收请求
  - 3.2: 生成临时文件ID
  - 3.3: 保存临时文件
  - 3.4: 生成可访问URL
  - 3.5: 返回结果

#### 步骤4：保存到数据库
- **位置**: `lobster_online/backend/app/api/assets.py` - `upload_asset()`
- **日志标记**: `[上传流程-步骤4]`
- **内容**: 记录asset_id和source_url

#### 步骤5：返回结果
- **位置**: `lobster_online/backend/app/api/assets.py` - `upload_asset()`
- **日志标记**: `[上传流程-步骤5]`
- **内容**: 记录最终状态（TOS成功 / 服务器临时文件；**仅当已有公网 URL 时到达**）

#### 失败（TOS 与 upload-temp 均未成功）
- **位置**: `lobster_online/backend/app/api/assets.py` - `upload_asset()`（步骤 4 之前）
- **日志标记**: `[上传流程-失败]`
- **行为**: 删除本次本地临时文件，`HTTP 503`，**不写数据库、不返回 asset_id**，不进入后续对话/图生视频链。

### 使用素材阶段

#### 步骤A：获取素材公网URL
- **位置**: `lobster_online/backend/app/api/assets.py` - `get_asset_public_url()`
- **日志标记**: `[使用素材-步骤A.1]` 到 `[使用素材-步骤A.4]`
- **子步骤**:
  - A.1: 查询素材
  - A.2: 检查source_url
  - A.3: 检测内部地址
  - A.4: 返回结果

#### 步骤B：处理附件URL列表（对话 / 图生视频注入前）
- **位置**: `lobster_online/backend/app/api/chat.py` - `_resolve_attachment_urls_strict()`（由 `_get_attachment_public_urls` 调用）
- **日志标记**:
  - `[使用素材-步骤B.1]` 每张附图开始
  - `[使用素材-步骤B.2]` `get_asset_public_url` 已得到公网 URL
  - `[使用素材-步骤B.5]` 全部附图 URL 齐全
  - `[使用素材-失败]` **无公网 source_url、或命中已禁止的 `/api/assets/file/` 签名链** → `HTTP 400`，**不调用 LLM/MCP**
- **说明**: 不再使用 `build_asset_file_url`；公网链仅来自上传阶段成功的 **TOS** 或 **upload-temp**。

#### 步骤B'：用户消息正文中的附图说明
- **位置**: `lobster_online/backend/app/api/chat.py` - `_build_user_content_with_attachments()`
- **规则**: 与步骤 B 相同数据源（仅 `get_asset_public_url`）；失败时同样 `HTTP 400`，日志 `[CHAT] 附图无公网 URL` / `[CHAT] 附图为已禁止的签名直链`

### 服务器端MCP处理阶段

#### 步骤C：检测和转存内部URL
- **位置**: `lobster-server/mcp/http_server.py` - `_call_tool()`
- **日志标记**: `[服务器端MCP-步骤C.1]` 到 `[服务器端MCP-步骤C.7]`

##### C.1-C.3: URL检测
- **C.1**: 开始检测URL
- **C.2**: URL解析结果
- **C.3**: 检测结果（内部地址/公网地址）

##### C.4-C.5: TOS转存（方法1）
- **C.4**: 确认为内部地址，开始转存
- **C.5**: 检查服务器端TOS配置
  - **C.5.1**: 开始下载内部图片
  - **C.5.2**: 下载响应状态
  - **C.5.3**: 下载成功
  - **C.5.4**: 开始上传到TOS
  - **C.5.5**: TOS转存结果

##### C.6: sutui.transfer_url转存（方法2）
- **C.6**: TOS转存未成功，尝试sutui.transfer_url
  - **C.6.1**: 调用sutui.transfer_url
  - **C.6.2**: 调用完成
  - **C.6.3**: 解析响应（多种方式）
  - **C.6.4**: 验证URL格式

##### C.7: 最终结果
- **C.7**: 转存成功/失败

## 日志级别说明

- **INFO**: 正常流程步骤
- **WARNING**: 非致命问题（如TOS未配置、检测到内部地址）
- **ERROR**: 错误情况（如HTTP错误、转存失败）

## 从上传到发出 video.generate 的节点顺序（本仓库 lobster_online）

| 顺序 | 日志前缀 / 位置 | 文件 |
|------|-----------------|------|
| 1 | `[上传流程-步骤1]` | `assets.py` `upload_asset` |
| 2 | `[上传流程-步骤2]`、`[TOS-步骤2.*]` | `assets.py` |
| 3 | `[上传流程-步骤3]` | `assets.py` → 服务器 `[服务器端-步骤3.*]` |
| 3a | `[上传流程-失败]` | 两路均失败，503，终止 |
| 4 | `[上传流程-步骤4]`、`[上传流程-步骤5]` | `assets.py` |
| 5 | `[使用素材-步骤A.*]` | `assets.py` `get_asset_public_url`（对话里解析附图时） |
| 6 | `[使用素材-步骤B.1]`～`B.2`、`B.5` 或 `[使用素材-失败]` | `chat.py` `_resolve_attachment_urls_strict` |
| 7 | `[CHAT] 注入素材 URL` | `chat.py` `_build_user_content_with_attachments` |
| 8 | `[CHAT] tool_call` / `invoke_capability` | `chat.py` |
| 9 | `[CHAT] 图生视频注入 filePaths` | `chat.py` `_inject_video_media_urls` |
| 10 | `[CHAT] 发 MCP video.generate 完整 payload` | `chat.py` `_exec_tool` |
| 11 | 本机 MCP → 服务器 MCP 上游 | `mcp/http_server.py`（另见服务器 `[服务器端MCP-步骤C.*]`） |

## 排查问题时的关键日志

1. **上传失败**: 查看 `[上传流程-步骤2]`、`[上传流程-步骤3]`、`[上传流程-失败]`
2. **TOS上传失败**: 查看 `[TOS-步骤2.*]` 日志
3. **服务器临时文件上传失败**: 查看 `[上传流程-步骤3]` 和 `[服务器端-步骤3.*]`
4. **内部地址检测**: 查看 `[使用素材-步骤A.3]`
5. **对话拒图（无公网链）**: `[使用素材-失败]`、`[CHAT] 附图无公网 URL`
6. **服务器端转存失败**: 查看 `[服务器端MCP-步骤C.*]` 日志

## 注意事项

- 所有日志都包含 `asset_id` 或 `url_key` 用于追踪
- URL在日志中会被截断到80-100字符，避免日志过长
- 错误日志包含完整的异常堆栈信息（`exc_info=True`）
