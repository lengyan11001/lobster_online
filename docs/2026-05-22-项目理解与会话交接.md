# 必火 AI 员工项目理解与会话交接

> 更新时间：2026-05-22  
> 目的：给新会话快速接手 `lobster_online` / `lobster_server`，避免重复摸索、误部署、误打包。

## 1. 仓库与职责边界

本机主要有两个仓库：

- `D:\lobster_online`
  - 用户电脑上的在线客户端。
  - 负责本机页面、OpenClaw、本机 MCP、素材库、发布账号、定时任务领取与执行、桌面 EXE 启动。
  - 静态页面在 `static/index.html`，主要 JS 在 `static/js/`。
  - 本机后端在 `backend/app/`，启动后通常监听 `127.0.0.1:8000`。
  - 本机 MCP 在 `mcp/http_server.py`，通常监听 `127.0.0.1:8001`。

- `D:\lobster_server`
  - 云端服务。
  - 负责登录注册、鉴权、积分/支付、云端配置、定时任务中心、短信、小程序 API、部分模型与能力网关。
  - 只有用户明确说“发布服务器/部署服务器”才部署。

基本原则：

- `online` 是用户机器执行面，`server` 是中心控制面。
- 用户没明确说发布服务器，不要部署。
- 用户没明确说打 OTA，不要打包。
- 用户多次强调：OTA 包和服务器发布是两个动作，只有明确说发布才发布。
- `lobster_online` 不整体上传服务器。

## 3. 本轮最新功能改动

### 3.1 定时任务新增「文案+创意图片」

用户要求：把原来「创意成片」前半段拆出来，只做到 AI 生成文案/图片提示词，再调用图片生成，生成到图片就结束。

已改：

- `static/js/scheduled-tasks.js`
  - 新增能力 `goal.image.pipeline`，显示名：`文案+创意图片`。
  - 该能力不展示参数。

- `backend/app/api/goal_video_pipeline.py`
  - 新增 `run_goal_image_pipeline(...)`。
  - 逻辑：规划 LLM -> `image.generate` -> 返回图片素材/链接。

- `backend/app/api/h5_chat_channel.py`
  - 定时任务领取后支持 `goal.image.pipeline`。
  - 会根据记忆生成每次不同角度的图片目标和发布文案。

### 3.2 素材库提升为顶部导航

用户要求：素材库从发布中心里提出来，和发布中心同级。

已改：

- `static/index.html`
  - 顶部导航新增 `素材库`。
  - `content-assets` 变为顶层 content block。
  - 发布中心内不再显示素材库 tab。

- `static/js/init.js`
  - 新增 `content-assets` 页面装饰。
  - 点击素材库导航调用 `initAssetLibraryView()`。

- `static/js/publish.js`
  - 新增 `initAssetLibraryView()`。

### 3.3 素材库上传失败修复

问题：本地素材上传接口用 `FormData`，但前端带了 `Content-Type: application/json`，导致 multipart boundary 丢失。

已改：

- `static/js/publish.js`
  - 新增 `_authHeadersNoContentType()`。
  - `/api/assets/upload` 上传本地文件时删除 `Content-Type`，让浏览器自动带 boundary。

### 3.4 创意成片备选素材组

用户要求：

- 每个图片素材可以设置为创意成片备选素材。
- 输入备选组名字，同名素材进入同一组。
- 后续补充：每个素材只能属于一个备选组；修改时覆盖上次备选组。
- 不要浏览器顶部 `prompt()`，要统一页面样式。
- 素材库增加按备选组查询。

已改：

- `backend/app/api/assets.py`
  - 新增 `creative_candidate_group` 单组字段读写语义，存在 `Asset.meta` 中。
  - 兼容老数组字段 `creative_candidate_groups`，但只取第一个。
  - 新增：
    - `GET /api/assets/creative-candidate-groups`
    - `POST /api/assets/{asset_id}/creative-candidate-groups`
  - `GET /api/assets` 支持 `creative_group` 查询参数。
  - 返回资产时同时返回：
    - `creative_candidate_group`
    - `creative_candidate_groups`

- `backend/app/create_app.py`
  - 新增启动迁移 `_migrate_assets_meta_column()`，给老 DB 补 `assets.meta` 列。

- `static/index.html`
  - 素材库筛选增加 `assetCreativeGroupFilter`。
  - 新增统一 modal：`assetCreativeGroupModal`。

- `static/js/publish.js`
  - 图片素材卡片新增 `设为创意备选`。
  - 点击后打开页面 modal，不用原生 `prompt()`。
  - 保存后覆盖素材原备选组。
  - 筛选区可以按备选组过滤。

### 3.5 创意成片改为从备选组随机取图

用户要求：原创意成片不再先生成图片，而是：

1. AI 根据记忆生成本次文案/视频提示词。
2. 从用户选择的某个备选素材组里随机取一张图片。
3. 用这张图调用视频生成。
4. 最后生成朋友圈/发布文案并返回 H5。

已改：

- `static/js/scheduled-tasks.js`
  - `创意成片` 参数区显示 `备选素材组` 下拉。
  - 下拉来自素材库的 `creative-candidate-groups`。
  - 创建任务时 payload 只传 `{ candidate_group }`。

- `backend/app/api/h5_chat_channel.py`
  - 新增 `_pick_creative_candidate_asset(...)`。
  - `goal.video.pipeline` 执行时要求 `candidate_group`。
  - 从本地素材库随机取该组图片，取到公网/签名 URL 后传给视频流水线。
  - 新增 `_run_goal_video_scheduled_pipeline(...)`。

- `backend/app/api/goal_video_pipeline.py`
  - 新增 `run_goal_video_from_reference_pipeline(...)`。
  - 逻辑：规划 LLM -> 直接用指定参考图调用 `video.generate`，跳过图片生成。

### 3.6 执行记录中输入素材和生成素材分开显示

用户反馈：备选图素材 ID 是 `49b218411b06`，但执行记录显示了别的素材 ID。

定位结果：

- 原 `素材：` 文案来自结果对象递归收集的 `final_asset_id/video_asset_id/image_asset_id`。
- 它展示的是生成结果素材，不是本次输入参考图。

已改：

- `backend/app/api/h5_chat_channel.py`
  - `_scheduled_complete_text(...)` 增加 `input_refs`。
  - 创意成片结果文本现在类似：

```text
生成完成。
发布文案：...
传给技能的提示词：...
备选组：美女生成
使用备选素材：49b218411b06
生成素材：xxxxxx
预览链接：
https://...
```

### 3.7 创意成片只返回最终视频链接

用户反馈：生成记录/手机消息/H5 返回两个链接。

定位结果：

- `_collect_scheduled_result_refs()` 递归捞所有 URL，会把参考图、上游临时链接、保存后素材链接都混进去。

已改：

- `backend/app/api/h5_chat_channel.py`
  - 新增 `_scheduled_goal_video_result_refs(...)`。
  - `goal.video.pipeline` 优先用 `video_asset_id/final_asset_id` 取本地保存后的最终素材 URL。
  - 如果没有素材 id，才兜底取一个视频 URL。
  - 最多返回 1 个视频链接。

- `static/js/scheduled-tasks.js`
  - 定时任务 tab 前端也过滤：`goal.video.pipeline` 只展示视频扩展名 URL，最多 1 个。
  - 这是为了旧记录 payload 里已有两个链接时，前端也不展示多余链接。

### 3.8 定时任务 tab 改版

用户要求：

- 原来 2 个 tab 改成 3 个：
  1. 创建新任务
  2. 执行中任务
  3. 执行结束任务
- 执行记录按状态拆到两个 tab。
- 三个 tab 必须放一行。

已改：

- `static/index.html`
  - 三个 tab 顺序：`创建新任务 | 执行中任务 | 执行结束任务`。
  - `schedTabCreate`：创建表单 + 任务列表。
  - `schedTabRunning`：执行中的 run 列表。
  - `schedTabFinished`：结束的 run 列表。
  - CSS 改成 3 等分一行，窄屏横向滚动不换行。

- `static/js/scheduled-tasks.js`
  - `runIsRunning(...)` 判断：
    - `pending`
    - `processing`
    - `running`
    - `queued`
    - `waiting`
  - 这些放入 `执行中任务`。
  - 其余状态放入 `执行结束任务`。

## 7. 打包和部署注意事项

### OTA

- 用户明确说“打 OTA 包”才打。
- OTA 包默认输出到 `lobster_online` 上一级目录，不要只在 `D:\lobster_online` 里找。
- `scripts/pack_client_code_ota.py` 是当前 OTA 脚本。
- CAB/WebView2 大依赖不要进普通 OTA，完整依赖包另走完整包。

常用检查：

```powershell
cd /d D:\lobster_online
python scripts\pack_client_code_ota.py
```

但只有用户明确要求时再执行。

### 服务器部署

- 用户明确说“发布服务器/部署服务器”才发布。
- 不要自动部署。
- 不需要发布海外，除非用户明确说。
- server 上有部署脚本，按 `D:\lobster_server` 里的文档/脚本走。

### git

- 用户明确说提交/推送才提交。
- 这个项目经常有别人同时改，先 `git pull` 合并再推。
- 不要回退别人改动。

如果要验证 UI：

3. 进入顶部 `素材库`。
4. 上传/保存一张图片素材。
5. 点 `设为创意备选`，输入组名。
6. 进入顶部 `定时任务`。
7. 在 `创建新任务` 中选择：
   - `创意成片`：选择备选素材组。
   - `文案+创意图片`：无参数。
8. 创建任务后切到：
   - `执行中任务`

- 创意成片结果显示：
  - `备选组`

## 9. 用户偏好和禁忌

- 用户希望直接定位、直接改，不喜欢泛泛解释。
- UI 要紧凑，不要一堆字段一行一个浪费空间。
- 能选择的数据不要让用户填 JSON。
- 不要新增“兜底/容错/硬编码限制”来掩盖设计问题。
- 不要把能力内部逻辑搬到定时任务里；定时任务只是固定入口和参数组织，实际能力仍走原执行入口。
- 扣费由能力内部处理，定时任务不处理扣费。
- 用户很在意：
  - 不要擅自部署。
  - 不要擅自打包。
  - 不要覆盖配置、支付、登录等环境配置。
  - 不要把样式同步到不该同步的目录。
