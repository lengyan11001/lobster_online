# 素材剪辑（media.edit）优化 — 跨版本同步说明

**权威实现**：`lobster_online` 本机后端（`media.edit` 走 `LOCAL_API_BASE`，实际执行在 `backend/app/services/media_edit_exec.py`）。

## 一、须同步的文件（以 `lobster_online` 为源）

| 路径 | 作用 |
|------|------|
| `backend/app/services/media_edit_exec.py` | **核心**：ffmpeg 叠字（含 CJK `fontfile`、overlay 全参数解析、`run_operation`） |
| `backend/app/api/assets.py` | **`_save_bytes_or_tos`**：剪辑产出写入素材库/TOS（若他仓缺此函数会导入失败） |
| `backend/app/api/media_edit.py` | 路由入口（一般随能力一并检查） |
| `backend/app/api/chat.py` | 系统提示里 `invoke_capability` / `overlay_text` 示例与文档引用 |
| `mcp/capability_catalog.json` | MCP/能力 `arg_schema`（`media.edit` 下 overlay 全字段） |
| `skill_registry.json` | 技能商店与前端可见的 `media.edit` schema |
| `skills/media_edit/merge_snippets/capability_catalog_media_edit.json` | 合并到能力目录用片段 |
| `skills/media_edit/merge_snippets/skill_registry_media_edit_skill.json` | 合并到技能注册用片段 |
| `deps/ffmpeg/README.txt` | ffmpeg / `LOBSTER_DRAWTEXT_FONT` / overlay 说明 |
| `docs/素材剪辑-overlay_text参数.md` | overlay 白名单字段说明 |
| `docs/文档与规则索引.md` | 索引中「overlay / 剪辑」条目（可选） |

## 二、按目标仓库怎么合

### 1. `lobster_online`（在线客户端完整包）

- 上述文件 **整体覆盖** 即可；再打 `pack_code.sh` / `pack_full_project.sh` 等。

### 2. `lobster-server`（云端鉴权 / 积分 / 技能元数据）

- **不部署** `media_edit_exec.py`（剪辑不在 ECS 跑，见架构文档）。
- **必须同步**：`skill_registry.json` 里 **`packages.media_edit_skill.capabilities.media.edit.arg_schema`**，与 `lobster_online/skill_registry.json` **保持一致**，避免商店/下发 schema 与客户端不一致。
- 若该仓另有能力目录 JSON，按你们流程把 `mcp/capability_catalog.json` 中 **`media.edit`** 段对齐（如有）。

## 三、一键包

在 `lobster_online` 根目录执行：

```bash
bash scripts/pack_media_edit_sync.sh
```

会在**上一级目录**生成 `media_edit_sync_*.zip`，内含 `lobster_online/...` 前缀路径及根目录 `README_MEDIA_EDIT_SYNC.txt`；在 **workspace 根目录**执行 `unzip -o media_edit_sync_*.zip` 覆盖即可。包内亦包含 `scripts/pack_media_edit_sync.sh` 便于留档。
