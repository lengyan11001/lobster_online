# 产品交付说明（维护者）

## 架构（必读摘要）

| 组件 | 职责 |
|------|------|
| **lobster_server** | 账号、鉴权、积分/支付、速推网关（`SUTUI_SERVER_TOKEN`）、公网 `upload-temp`、Webhook 入口 |
| **lobster_online（用户本机）** | 静态页 + 本机后端：对话/OpenClaw、本机 MCP、素材、发布；JWT 由 server 签发，本机通过 `AUTH_SERVER_BASE` 校验 |

前端双入口：**`API_BASE`** → server；**`LOCAL_API_BASE`** → 本机后端（对话与本地能力）。

完整叙述：[架构说明_server与本地职责.md](./架构说明_server与本地职责.md)、[在线版架构-对话与OpenClaw.md](./在线版架构-对话与OpenClaw.md)。服务端部署：`../lobster-server/docs/云服务器部署说明.md`。

## 服务器发布（开发机）

```bash
cd lobster-server
chmod +x scripts/deploy_publish.sh scripts/deploy_server.sh scripts/deploy_from_local.sh
bash scripts/deploy_publish.sh
```

有未提交改动会自动 `git commit` 再 `push`，再 SSH 拉取并重启（依赖 `.env.deploy`）。若无需自动提交，只推送已 commit 的代码：`bash scripts/deploy_server.sh`。

## 客户端发布（代码包）

```bash
cd lobster_online
./pack_code.sh
```

产物：`lobster_online_code_*.zip`（内含 `pack_bundle.env`→`.env`、`README-一键使用.txt`、`一键启动.bat`）。用户侧**不要求改配置**。

网络地址实现约定：[架构-网络地址约定.md](./架构-网络地址约定.md)。
