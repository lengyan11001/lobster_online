# 全量模型清单 + 参数审计（给 Cursor / CI 复用）

目标：从速推 **实时** 拉取模型与 `params_schema`，再对照 **`mcp/http_server.py`** 里 `_normalize_image_generate_payload` / `_normalize_video_generate_payload` 的输出，发现必填缺失、枚举不合法、类型不一致等。

## 第一步：只抓全量模型清单（最快）

在 **`lobster_online`** 目录：

```bash
cd /path/to/lobster_online
export SUTUI_API_BASE=https://api.xskill.ai   # 可选
export XSKILL_API_KEY=sk-xxx                  # 若接口返回 401 再设；也可用 SUTUI_SERVER_TOKEN 同名语义
python3 scripts/xskill_fetch_models.py
```

- 默认写出 **`docs/_generated/xskill_models_list.json`**（含 `count` 与 `models` 数组）。
- 需要连同 **每个模型的 /docs** 一并快照时：

```bash
python3 scripts/xskill_fetch_models.py --schemas -o docs/_generated/xskill_models_with_schemas.json
```

脚本在 **HTTP 非 2xx** 或 **JSON 结构不符合预期** 时 **直接非零退出**（无静默兜底）。

## 第二步：81 个模型逐个按 schema 校验 normalize（与仓库已有脚本一致）

完整审计（并发拉 `params_schema` + 探测 payload + 校验 normalize 输出）：

```bash
cd /path/to/lobster-server
export XSKILL_API_KEY=sk-xxx   # 建议设置，与官方 xskill_api.py 一致
python3 scripts/audit_model_params.py
python3 scripts/audit_model_params.py -o docs/audit_report_latest.md
python3 scripts/audit_model_params.py --limit 5    # 调试：只跑前 5 个
```

说明见 **`lobster-server/docs/参数审计方法对比.md`**。

审计脚本会从 **`lobster-server/mcp/http_server.py`** `import` 两个 normalize 函数；若你改的是 **`lobster_online`** 里的同名函数，需**两边**保持同步后再跑，或在 `lobster_online` 下增加相同 import 路径（当前以 **server 为单一对照源**）。

## 与「直连 MCP」的关系

速推 **`generate` / `get_result`** 在 Lobster 内已统一走 **`/api/v3/tasks/create` 与 `tasks/query`（REST）**，与官方 `xskill_api.py` 的 `submit_task` 一致；审计的是 **normalize → 作为 `params` 发往 create** 的这一段，与是否走 MCP HTTP 无关。

## 产物位置

| 产物 | 说明 |
|------|------|
| `lobster_online/docs/_generated/xskill_models_list.json` | 清单快照（`xskill_fetch_models.py`） |
| `lobster_server/audit_report.md` 或 `-o` 指定 | 全量审计 Markdown |

可将 `docs/_generated/` 加入 `.gitignore` 若不想提交快照。
