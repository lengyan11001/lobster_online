# 轻盈 Agent 首页改版方案

## 目标
把当前偏“多面板工作台”的首页，改成更适合普通人使用的 AI 员工首页。

目标用户：
- 普通用户
- 想直接让 AI 帮自己干活的人
- 重点任务包含视频生成、电商套图、自动上架、小红书运营、抖音运营发布

设计倾向：
- 参考豆包的轻盈亲和感
- 不是后台管理系统风格
- 不是厚重的专业工作台
- 强调“先提需求，再触发能力”

## 现状问题
当前首页的主要问题不是功能不够，而是信息结构太重：

1. 主界面像系统后台，不像 AI 助手入口。
2. 左中右信息都比较重，用户第一眼不知道应该先做什么。
3. 很多能力藏在不同面板和 tab 里，普通用户理解成本高。
4. 当前视觉偏深色技术感，与“AI 员工帮你干活”的亲和定位不一致。
5. 聊天、技能、任务、产物之间缺少自然过渡，容易让人觉得功能是拼起来的。

## 总体改版原则
1. 首页只保留一个核心焦点：输入任务。
2. 高频功能做成底部标签，不强迫用户先进二级页面。
3. 右侧不做常驻重信息栏，避免分散注意力。
4. 任务执行过程改成“消息流 + 状态卡片”的形式，弱化后台感。
5. 参数配置尽量后置，先让用户开口，再在需要时补素材、补设置。
6. 旧页面不消失。技能商店、发布、系统配置、代理商等页面继续保留，只是重做入口层级与视觉风格。
7. 面向桌面宽屏优化。不能照搬豆包那种窄中栏布局，要让内容区在 1440px-1680px 视口下更舒展。

## 新的约束条件
这次首页改版不是重新发明一个纯聊天产品，而是在保留现有系统能力的前提下重做入口层。

明确约束：
- 技能商店要继续展示，而且入口不能藏太深
- 现有旧页面仍需保留，只是视觉和信息层级要重整
- 首页宽度不能太窄，尤其要适配桌面端大屏使用
- 豆包只参考“轻盈亲和感”和“底部功能标签”，不直接照搬其窄版聊天中轴

因此，这次设计目标应该调整为：
- 首页更像“轻盈版 AI 工作台”
- 而不是“只有一个聊天框的极简页”

## 方案 A：豆包式中心输入页

### 适合场景
最适合作为默认首页，也是最接近你提到的方向，但需要改造成“宽屏版”。

### 核心思路
首页就是一个轻盈的 AI 助手欢迎页，重点放在中央输入区和底部能力标签。用户先说需求，再根据需要点击标签补充“做视频、做套图、自动上架”等能力。

### 页面结构
1. 左侧窄导航
2. 中间宽屏欢迎区 + 主输入框
3. 输入框下方功能标签区
4. 中下区域展示最近任务、推荐模板、快捷入口
5. 技能商店和旧页面通过一级导航持续可达

### 线框草图
```text
+----------------------------------------------------------------------------+
| Logo | 新对话 | 技能商店 | 发布中心 | 系统配置 | 历史任务                   |
|------|---------------------------------------------------------------------|
| 导航 |                        你好，今天想让我做什么？                      |
|      |            [ 在这里输入你的需求......                           ]    |
|      |            [ 上传素材 ] [ 语音 ] [ 发送 ]                           |
|      |                                                                     |
|      |   [ 图像生成 ] [ 视频生成 ] [ 电商套图 ] [ 自动上架 ] [ 小红书运营 ] |
|      |   [ 抖音发布 ] [ 店铺运营 ] [ 技能商店 ] [ 更多 ]                   |
|      |                                                                     |
|      |   最近任务                     常用工具                              |
|      |   [ 昨天帮你做的视频 ]        [ 技能商店 ] [ 发布中心 ] [ 素材库 ]   |
|      |   [ 已生成套图 ]              [ 账号运营 ] [ 系统配置 ]              |
+----------------------------------------------------------------------------+
```

### 优点
- 最容易让普通用户看懂
- 首页转化效率高，第一步动作非常明确
- 很适合后续接“输入一句话直接开始干活”的体验
- 能保留现有系统功能，不会让旧能力消失

### 风格建议
- 浅暖灰背景
- 白色或微暖白输入区
- 低饱和品牌色作为点缀
- 大标题友好，但不过度营销
- 标签胶囊化，像“会的技能”而不是“系统模块”

### 推荐程度
最高。建议作为主方案。

## 方案 B：场景卡片式 AI 员工首页

### 适合场景
适合强调“它到底能帮我做什么”，比方案 A 更强功能引导。

### 核心思路
中间依然保留主输入框，但下方不是单纯标签，而是几张大场景卡片，直接对应“做视频”“做电商图”“帮我上架”“做内容运营”。

### 页面结构
1. 左侧导航保留
2. 中间主输入框
3. 下方四到六张场景卡片
4. 更下面再展示最近任务

### 线框草图
```text
+----------------------------------------------------------------------------+
| 导航 |                   你的 AI 员工，今天想做什么？                        |
|      |          [ 描述任务，或选择下面一个场景开始 ]                        |
|      |                                                                     |
|      |   +--------------+ +--------------+ +--------------+                |
|      |   | 做视频       | | 电商套图     | | 自动上架     |                |
|      |   | 口播/混剪    | | 主图/详情图  | | 发布到店铺   |                |
|      |   +--------------+ +--------------+ +--------------+                |
|      |   +--------------+ +--------------+ +--------------+                |
|      |   | 小红书运营   | | 抖音运营     | | 技能商店     |                |
|      |   +--------------+ +--------------+ +--------------+                |
|      |                                                                     |
|      |   下方继续展示最近任务 / 最近发布 / 常用页面快捷入口                  |
+----------------------------------------------------------------------------+
```

### 优点
- 用户不用猜能干什么
- 适合新用户首次进入
- 对“AI 员工”定位很直观

### 风险
- 容易比豆包风格更重一点
- 如果卡片做太多，会又回到当前“功能很多”的问题

### 推荐程度
中高。适合作为首页首屏或欢迎态，不适合作为长期复杂工作台。

## 方案 C：对话主导 + 底部技能条

### 适合场景
适合把产品更明确地做成“会执行任务的聊天助手”，比方案 A 更接近 OpenAI/豆包混合风格。

### 核心思路
首页打开就是对话流，但不做传统空白聊天页，而是在输入框底部提供一排可切换的能力标签。用户输入后，系统在消息流中以“任务卡”的方式呈现生成、上架、发布等动作。

### 页面结构
1. 左侧是新对话和历史记录
2. 中间是消息流
3. 底部是固定输入框 + 技能条
4. 顶部补充系统级导航，承接技能商店、发布中心、系统配置等旧页面
5. 右侧不常驻，只有需要时从抽屉中展开“任务详情”

### 线框草图
```text
+----------------------------------------------------------------------------+
| 导航 | 新对话 | 技能商店 | 发布中心 | 系统配置                              |
|      |----------------------------------------------------------------------|
|      | AI：我可以帮你做视频、套图、上架和内容运营                           |
|      | 你：帮我做一条女装带货视频                                           |
|      | AI：[任务卡] 已识别为“抖音视频生成”                                  |
|      | AI：[任务卡] 请上传商品图或继续补充要求                              |
|      |                                                                      |
|      |----------------------------------------------------------------------|
|      | [ 输入你的需求...                                              ]     |
|      | [ 图像生成 ][ 视频生成 ][ 电商套图 ][ 自动上架 ][ 技能商店 ]         |
+----------------------------------------------------------------------------+
```

### 优点
- 最像“AI 员工”在持续协作
- 功能与执行结果能自然融合进聊天流
- 很适合你后面的 agent 化功能扩展

### 风险
- 需要对消息流组件和任务卡组件做更系统的重构
- 比纯首页美化更像产品层升级

### 推荐程度
高。如果你后面重点做 agent 执行，这套会最耐用。

## 三套方案对比

| 方案 | 体验关键词 | 上手难度 | 扩展性 | 推荐用途 |
|------|------------|----------|--------|----------|
| A | 轻盈、友好、豆包感最强 | 最低 | 中高 | 默认首页 |
| B | 功能清晰、场景导向 | 低 | 中高 | 新用户欢迎页 |
| C | 对话协作、agent 感最强 | 中 | 最高 | 长期主产品结构 |

## 宽度与布局建议

这次一定不能再用过窄的中间列。建议采用下面的桌面布局标准：

- 页面总容器：`max-width: 1480px` 到 `1680px`
- 左侧导航：`240px` 到 `280px`
- 中央主工作区：至少 `900px+`
- 内容区采用 `padding-inline: 32px ~ 48px`
- 聊天页欢迎态不要把输入框压成一条窄柱，而要让它接近“横向工作台”

推荐视觉结构：

1. 左侧保留全局导航和历史记录
2. 顶部保留系统级 tab，承接技能商店、发布、配置等旧能力
3. 中间主区承担欢迎态、对话态、任务态
4. 右侧只在必要时以抽屉或浮层形式出现，不常驻

## 推荐的最终结构

综合你的补充，我现在最推荐的不是纯豆包式，而是：

### “宽屏轻盈版 Agent 工作台”

结构如下：
- 左侧：全局导航 + 历史记录
- 顶部：技能商店 / 发布中心 / 系统配置 / 账号运营等系统入口
- 中间首屏：豆包式欢迎文案 + 大输入框
- 输入框下：功能标签
- 首屏下半区：最近任务 + 常用功能快捷入口
- 任务执行时：进入对话流 + 任务卡

这个结构能同时满足：
- 普通用户一进来知道先说需求
- 老功能不会丢
- 大屏不会显得空和窄
- 后续做 agent 化扩展也有空间

## 最新定稿约束

基于最新讨论，这一版首页设计额外固定以下规则：

1. 左侧主导航以“历史对话” 为核心，不做右侧常驻信息栏。
2. 右上角保留小头像或账户菜单入口，继续承接登录信息、账户设置、退出登录。
3. 技能商店、发布中心、系统配置、代理商等旧页面继续保留，并通过顶部系统级导航进入。
4. 首页中间区域必须足够宽，不做窄栏式聊天页。
5. 输入框下方保留“功能标签”作为核心快捷入口。

## 关键区域设计

### 1. 顶部系统导航

#### 组成
- 左上：品牌 Logo
- 中间：系统级 tab
- 右上：小头像/菜单入口

#### 建议导航项
- 首页
- 技能商店
- 发布中心
- 系统配置
- 账号运营

#### 设计重点
- 顶部不要太高，建议 56px-64px
- 用轻量 tab 风格，不做厚重按钮条
- 右上角头像只做轻入口，不抢主视觉

### 2. 左侧历史对话区

#### 组成
- 新对话按钮
- 搜索框
- 历史对话列表
- 可选固定分组：今天、最近 7 天、更早

#### 设计重点
- 左栏宽度建议 260px 左右
- 比当前更像“会话导航”，减少后台感
- 当前会话要有清晰高亮
- 历史项支持两行：标题 + 时间/摘要

### 3. 中间欢迎区

#### 欢迎文案建议
- 主标题：`今天想让我帮你做什么？`
- 副标题：`做视频、出电商图、自动上架、运营账号，都可以直接交给我。`

#### 设计重点
- 不能像现在一样被多个 panel 切碎
- 保持居中，但宽度要足够大
- 标题友好，不要太“技术平台化”

### 4. 主输入框

#### 交互建议
- 默认是一块大输入框
- 支持上传图片/视频
- 支持语音按钮
- 支持点击功能标签后自动补全引导词

#### 占位文案建议
- `输入需求，比如：帮我生成一条女装抖音带货视频`
- `也可以上传商品图，让我直接开始做`

#### 设计重点
- 输入框建议做成轻卡片，不要像表单 textarea
- 宽度建议 820px-980px
- 高度建议 120px 左右，支持多行自然输入

### 5. 功能标签区

#### 推荐标签
- 视频生成
- 电商套图
- 自动上架
- 小红书运营
- 抖音发布
- 图像生成
- 技能商店
- 更多

#### 设计重点
- 标签做胶囊按钮
- 不同功能可用轻微色差区分，但整体仍要克制
- 点击后不是跳复杂页面，而是优先把意图带入输入框

### 6. 最近任务 + 常用功能区

#### 最近任务
展示最近完成或进行中的任务，如：
- 最近生成的视频
- 最近电商套图
- 最近发布记录

#### 常用功能
展示高频页面快捷入口，如：
- 技能商店
- 发布中心
- 账号运营
- 素材库

#### 设计重点
- 这块不能做得像后台报表
- 更像“继续上次工作”和“快速开始”

### 7. 右上角头像菜单

#### 保留内容
- 用户昵称/邮箱
- 当前算力或套餐信息
- 账户设置
- 退出登录

#### 设计重点
- 保留现有登录逻辑
- 视觉更轻，像产品菜单，不像系统弹框
- 头像建议用圆形小尺寸，32px-36px 即可

## 任务执行态设计

首页是欢迎态，真正开始执行任务后，页面进入“对话主导态”。

### 执行态结构
- 左侧：历史对话不变
- 顶部：系统级导航不变
- 中间：消息流 + 任务卡
- 底部：输入框 + 功能标签
- 右上角：头像菜单不变

### 任务卡类型
- 正在理解需求
- 正在生成图片
- 正在生成视频
- 正在自动上架
- 正在发布到抖音/小红书
- 已完成，可查看结果

### 任务卡视觉
- 白底轻描边
- 左侧小状态点或图标
- 标题清晰
- 附加描述一句话说明进展
- 结果卡可以直接显示缩略图/视频封面

---

## 视觉规范建议

### 页面基调
- 浅色、柔和、整洁
- 不走黑底科技风
- 保持亲和，但不做幼态卡通化

### 色彩
- 主背景：暖灰白
- 面板：纯白或微暖白
- 主强调色：柔和蓝
- 功能标签辅色：橙 / 绿 / 红 / 蓝做轻区分

### 阴影和边框
- 阴影很轻
- 边框浅灰
- 圆角偏柔和，但不夸张

### 字体层级
- 标题明显更大
- 辅文更淡
- 标签和状态信息字重更稳

## 实施优先级

### 第三阶段

- 加最近任务快捷恢复
- 加更细的空态/加载态/失败态设计

## 我建议的落地路径
最稳的做法不是三选一硬切，而是组合：

1. 首页首屏采用方案 A
2. 首页下半屏融入方案 B 的场景卡片
3. 真正进入执行后，任务页采用方案 C 的对话主导结构

这样做的好处是：
- 首次进入足够轻盈
- 新用户能快速理解能力边界
- 进入执行阶段后，又不会因为结构太轻而承载不了复杂任务

## 视觉风格建议

### 颜色
- 主背景：暖白或极浅灰白
- 卡片背景：白色
- 强调色：偏柔和的蓝色或青蓝，但饱和度比当前低很多
- 辅助色：橙色、绿色、红色仅用于不同功能标签，不大面积使用

### 组件
- 输入框做大，像欢迎用户说话，而不是像表单提交框
- 功能入口优先使用胶囊标签和轻卡片
- 历史记录弱化分割线，强调可扫描性
- 任务状态使用温和色块，不用高压警示色

### 不建议继续保留的做法
- 大面积深色背景
- 厚重描边卡片套卡片
- 默认同时显示大量参数表单
- 常驻右侧大信息栏
