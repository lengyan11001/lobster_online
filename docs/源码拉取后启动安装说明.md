# 源码拉取后启动安装说明

适用场景：开发同事直接 `git clone` / `git pull` 了 `lobster_online`，不是从“完整版安装包”解压出来的目录。

先说结论：**源码仓库不是完整运行包**。完整包里会带 `python/`、`nodejs/`、`deps/wheels/`、`browser_chromium/`、`deps/ffmpeg/` 等大依赖；普通 git 拉代码通常没有这些目录或目录不完整，所以直接双击 `install.bat` 可能装不起来。

## 1. 先确认自己拿到的是哪种目录

在 `lobster_online` 根目录看这些路径：

```bat
dir python\python.exe
dir nodejs\node.exe
dir nodejs\node_modules\openclaw
dir nodejs\node_modules\@tencent-weixin\openclaw-weixin
dir deps\wheels
dir browser_chromium
dir deps\ffmpeg\ffmpeg.exe
```

如果这些大多存在，就是接近完整包；如果不存在，就是纯源码。

也可以用项目自带只读检查：

```bat
python scripts\report_pack_gaps.py
```

这个脚本只检查缺口，不下载、不改文件。看到 `MISSING` 就说明对应依赖没带齐。

## 2. 纯源码机器推荐安装方式

纯源码机器优先跑：

```bat
install_slim.bat
```

`install_slim.bat` 是给“无完整版依赖目录”的机器用的，会在线补：

- Python 3.12
- pip
- Python requirements
- VC++ Runtime
- Node.js
- OpenClaw
- Playwright Chromium
- ffmpeg

这台机器必须能访问 python.org、nodejs.org、PyPI、npm、Playwright 下载源等。如果公司网络拦截，需要配置代理或改用完整包。

安装后启动：

```bat
start.bat
```

## 3. 完整包目录安装方式

如果目录里已经有 `python/`、`nodejs/`、`deps/` 等完整依赖，使用：

```bat
install.bat
start.bat
```

`install.bat` 的逻辑是优先用包内离线资源；包内缺某些东西且没有设置纯离线时，会尝试联网补。

如果要强制纯离线，不允许任何下载：

```bat
set LOBSTER_OFFLINE_ONLY=1
install.bat
```

这种模式下必须提前带齐 `deps/wheels/`、`python/`、`nodejs/`、`browser_chromium/`、`deps/ffmpeg/` 等，否则会失败或跳过对应能力。

## 4. 必装依赖怎么逐项看

### Python

优先级：

1. `python\python.exe`
2. 系统 `python`
3. `install_slim.bat` 在线安装 Python 3.12

检查：

```bat
python --version
python -m pip --version
```

如果用包内 Python：

```bat
python\python.exe --version
python\python.exe -m pip --version
```

### Python 包

来源：

- `requirements.txt`
- 完整锁定参考：`requirements-full.txt`

安装：

```bat
python -m pip install -r requirements.txt
```

关键模块检查：

```bat
python -c "import fastapi,uvicorn,pydantic,httpx,sqlalchemy,playwright,greenlet,PIL"
```

完整功能还会用到：

```bat
python -c "import tos, Crypto, googleapiclient, httplib2, twilio"
```

### 离线 wheels

完整包要求 `deps\wheels` 能覆盖 `requirements.txt`。

检查：

```bat
python scripts\verify_offline_wheels.py
```

补齐 wheels 需要在有网机器执行：

```bat
python scripts\prepare_offline.py --target windows
python scripts\verify_offline_wheels.py
```

### VC++ Runtime

Windows 上 `greenlet` 等原生 wheel 可能依赖 VC++ 2015-2022 x64。

完整包建议带：

```text
deps\vc_redist.x64.exe
```

如果本机已安装，可跳过：

```bat
set LOBSTER_SKIP_VCREDIST=1
install.bat
```

### Node.js / OpenClaw

OpenClaw 需要 Node。

检查：

```bat
node --version
dir nodejs\node.exe
dir nodejs\node_modules\openclaw
dir nodejs\node_modules\@tencent-weixin\openclaw-weixin
```

纯源码联网安装时，`install_slim.bat` 会补 Node 和 OpenClaw。

如果已有 `nodejs\package.json`，可手动：

```bat
cd nodejs
npm install
cd ..
python scripts\setup_openclaw.py
```

`setup_openclaw.py` 会生成/修复 `openclaw\openclaw.json`，并注册微信 OpenClaw 插件。

### Playwright Chromium

发布、浏览器自动化能力需要 Chromium。

检查：

```bat
dir browser_chromium
python -m playwright install chromium
```

完整包可带 `browser_chromium/`。启动时 `start.bat` 会自动设置：

```bat
PLAYWRIGHT_BROWSERS_PATH=当前目录\browser_chromium
```

### ffmpeg

剪辑类能力需要：

```text
deps\ffmpeg\ffmpeg.exe
```

检查：

```bat
dir deps\ffmpeg\ffmpeg.exe
```

有网可补：

```bat
python scripts\ensure_ffmpeg_windows.py
```

### 配置文件

第一次安装会从 `.env.example` 复制 `.env`。

必须检查：

```bat
dir .env
type .env
```

常见需要配置的项：

- API Key / Token
- DeepSeek / OpenClaw 相关 Key
- HiFly Token
- 速推 Token
- TOS / CDN 配置
- 端口：`PORT=8000`、`MCP_PORT=8001`

注意：`.env` 是本机配置，不应该直接提交到 git。

## 5. 启动后怎么判断服务正常

启动：

```bat
start.bat
```

它会启动：

- MCP：默认 `8001`
- Backend：默认 `8000`
- 浏览器打开 `http://localhost:8000`

检查端口：

```bat
netstat -ano | findstr ":8000"
netstat -ano | findstr ":8001"
```

看日志：

```bat
type backend.log
type mcp.log
type logs\app.log
```

常见错误：

- `uvicorn not installed`：Python 包没装好，重新跑 `install_slim.bat` 或 `python -m pip install -r requirements.txt`
- `Python not found`：没有系统 Python，也没有 `python\python.exe`
- `Node.js not found`：没有系统 Node，也没有 `nodejs\node.exe`
- `playwright chromium not installed`：执行 `python -m playwright install chromium`
- `ffmpeg.exe missing`：执行 `python scripts\ensure_ffmpeg_windows.py`
- `DeepSeekAPI Key无效或未配置`：检查 `.env` 和 `openclaw\.env`，再执行 `python scripts\setup_openclaw.py`

## 6. 给别人机器的建议流程

如果只是开发验证，推荐：

```bat
git clone <repo> lobster_online
cd lobster_online
install_slim.bat
start.bat
```

如果机器没网，别让他直接拉源码跑；应该给完整包，或者先在有网机器补齐依赖：

```bat
python scripts\report_pack_gaps.py
bash scripts\sync_deps_for_pack.sh
python scripts\report_pack_gaps.py
```

确认核心项没有 `MISSING` 后，再整体拷贝目录或打完整包。

## 7. online 和 server 的区别

- `lobster_online`：用户本机运行，负责本地执行、OpenClaw、发布、素材、浏览器自动化等。
- `lobster_server`：云端服务，部署到服务器，用 `requirements.txt` + systemd / 部署脚本启动。

本说明主要针对 `lobster_online` 本机源码安装。云端 `lobster_server` 不要用这里的 `install.bat/start.bat`，看 `lobster_server/README-部署.md` 和 `scripts/server_install.sh`、`scripts/server_update_and_restart.sh`。
