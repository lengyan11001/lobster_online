# 发布账号接口使用说明

本文档给其他技能或自动化流程使用：在发布前先获取可用账号，再按账号名或内部 id 发起发布。

## 获取账号列表

请求本地 online 后端：

```http
GET /api/accounts
Authorization: Bearer <用户登录 token>
```

返回结构：

```json
{
  "accounts": [
    {
      "id": -1001,
      "platform": "douyin",
      "platform_name": "抖音",
      "nickname": "抖音账号1",
      "status": "online",
      "is_origin_slot": true,
      "origin_account_id": 1
    },
    {
      "id": 12,
      "platform": "xiaohongshu",
      "platform_name": "小红书",
      "nickname": "小红书账号A",
      "status": "online"
    }
  ],
  "platforms": []
}
```

## 字段含义

`id`：接口内部使用的账号 id。普通账号来自 `publish_accounts.id`；抖音固定槽位是虚拟 id。

`platform`：平台标识，例如 `douyin`、`xiaohongshu`、`toutiao`。

`nickname`：用户看到和指定的发布账号名字。其他技能要展示给用户时，直接展示这个字段。

`status`：账号状态，常见为 `online`、`offline`、`pending`。

`is_origin_slot`：是否为抖音固定槽位。

`origin_account_id`：抖音固定槽位编号，`1`、`2`、`3`。

## 抖音固定槽位

抖音获客中心的 3 个固定槽位不是数据库真实行，而是从 `data/config.json` 的 `douyin_accounts` 映射出来：

```text
抖音账号1 -> id = -1001
抖音账号2 -> id = -1002
抖音账号3 -> id = -1003
```

其他技能不要自己拼这些 id，应该以 `/api/accounts` 返回值为准。

## 发布调用建议

如果调用本地后端 `/api/publish`，可以传 `account_id`，也可以传 `account_nickname`。为了兼容抖音固定槽位和普通账号，推荐统一传 `account_nickname`：

```http
POST /api/publish
Authorization: Bearer <用户登录 token>
Content-Type: application/json
```

```json
{
  "asset_id": "3c5204e6421b",
  "account_nickname": "抖音账号1",
  "title": "作品标题",
  "description": "发布文案",
  "tags": "AI,营销"
}
```

如果是 MCP / 对话工具里的 `publish_content`，必须传 `account_nickname`，不要传数据库 id：

```json
{
  "asset_id": "3c5204e6421b",
  "account_nickname": "抖音账号1",
  "title": "作品标题",
  "description": "发布文案"
}
```

## 推荐流程

1. 调用 `GET /api/accounts`。
2. 按 `platform` 过滤目标平台。
3. 展示 `nickname` 给用户选择。
4. 发布时把用户选择的 `nickname` 原样传给 `account_nickname`。
5. 不要写死账号名，不要把用户口头说的数字直接当数据库 id。
